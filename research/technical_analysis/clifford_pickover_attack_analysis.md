## 3. Data Poisoning Attack Analysis

### What is Data Poisoning?

**Definition:**
- Malicious manipulation of training data for ML/AI systems
- Injecting corrupted, biased, or malicious data
- Causing models to learn wrong patterns
- **Backdoor or degraded performance**

**Attack Types:**
1. **Availability Attacks** - Reduce model accuracy
2. **Integrity Attacks** - Insert backdoors
3. **Targeted Poisoning** - Specific outputs for specific inputs
4. **Model Replacement** - Replace entire model
